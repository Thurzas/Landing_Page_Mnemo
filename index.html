DESIGN · Déclenchement
Double déclenchement + EvaluationCrew séparée
Le crew unique evaluate+retrieve+main rendait impossible toute interception entre l'évaluation et le retrieval. Impossible de confirmer la query web avant que le LLM ne parte chercher.
EvaluationCrew séparée (config evaluation_tasks.yaml) + ConversationCrew pour retrieve+main. Interception propre dans handle_message() entre les deux kickoffs. needs_web + web_query dans le JSON évaluateur
FEATURE · Sécurité
Confirmation utilisateur — query figée
Sans garde-fou, le LLM pouvait déclencher des requêtes web avec des données personnelles, les rejouer plusieurs fois, ou modifier la query en cours de route.
_confirm_web_search() dans main.py : affiche backend + query sanitisée avant envoi. Si refus → needs_web=False injecté, le LLM répond depuis la mémoire. Query figée au moment de l'évaluation — immuable ensuite
FEATURE · web_tools.py
WebSearchTool — gardes-fous intégrés
Un point d'entrée vers l'extérieur sans protection = risque de fuite PII, prompt injection via résultats, ou accès à des URLs internes forgées.
Sanitisation query (email/IP/téléphone FR redactés, 200 chars max), blocklist URLs privées (RFC1918 + localhost + host.docker.internal), extraits tronqués à 400 chars marqués [SOURCE WEB], audit log local de chaque requête
FIX · DDG + Télémétrie
4 bugs post-déploiement corrigés
duckduckgo_search renommé en ddgs → RuntimeWarning. web_query trop littérale → DDG cherchait "résumé" en chinois. max_iter=5 trop juste pour 4 outils. CrewAI envoyait des traces vers app.crewai.com.
Import ddgs, prompt web_query avec exemples anglais/techniques, max_iter=8 sur le retriever, OTEL_SDK_DISABLED=true + CREWAI_DISABLE_TELEMETRY=true dans .env

## Sources web dans memory.md

Philosophie

Conçu pour durer.
Pas pour impressionner.

Chaque décision d'architecture a été prise pour la robustesse à long terme : mémoire éditable manuellement, zéro vendor lock-in, tests avant features.

📄
memory.md comme source de vérité
La mémoire reste un fichier Markdown lisible et éditable humainement. Versionnable avec git. SQLite est un index — pas la source. Les champs vides contiennent des notes d'intention pour l'agent.
🔒
100% local, 100% privé
Ollama en Docker, SQLite sur disque, aucun appel vers un service externe. Ce qui est dit à Mnemo reste sur la machine.
🧪
Python avant LLM
Réduire la surface LLM. La détection de gaps, l'écriture des réponses du questionnaire, la validation des sections — tout ce qui peut être fait en Python pur ne passe plus par un agent. Moins d'hallucinations, résultat garanti.
🧠
Mémoire ≠ base de données
La mémoire personnelle et les documents ingérés sont séparés physiquement mais fusionnés dans le retrieval — pour que l'agent sache toujours d'où vient chaque information.
Architecture

Qualité · Total cumulé

11 vrais bugs réglés.
Aucun inventé.

Phase 1 : 6 bugs de production détectés par les tests. Session d'aujourd'hui : 5 bugs supplémentaires découverts à l'usage.

BUG #1
Perte silencieuse de chunks
Le dernier ### de chaque ## dans memory.md était perdu lors du parsing. Des sous-sections entières n'étaient jamais indexées en SQLite.
Sauvegarde du chunk courant avant reset à chaque changement de section
BUG #2
Crash sur query vide
SQLite FTS5 lève OperationalError sur une string vide. Pipeline entier plantait si l'évaluateur produisait une memory_query vide.
Guard if not query.strip(): return [] en tête de search_keyword()
BUG #3
Collision de chunk_id
Hash MD5 sur le contenu seul : deux passages identiques dans un document produisaient le même ID — le second était silencieusement skippé.
Hash sur doc_id:page:chunk_index:contenu — chaque position unique
BUG #4
AttributeError para.style
Paragraphes créés par copier-coller dans Word peuvent avoir style=None. Crash immédiat à l'ingestion de tout DOCX réel.
Guard triple para.style and para.style.name and ...
BUG #5
Fonctions courtes disparaissent
Seuil MIN_CHUNK_LEN=80 filtrait des fonctions valides. Des pans entiers du code n'étaient pas indexés.
Seuil désactivé pour le code — la structure sémantique prime sur la longueur
BUG #6
Doublons AST walk()
ast.walk() descend récursivement — les méthodes de classes étaient indexées deux fois, causant du bruit dans le retrieval.
Remplacé par ast.iter_child_nodes() — niveau module uniquement
SESSION #7
Import GapDetectionCrew inexistant
main.py importait deux noms qui n'ont jamais existé dans crew.py. Résidu d'une version intermédiaire. Plantait immédiatement.
Remplacé par CuriosityCrew partout
SESSION #8
_TeeStream cassait input()
isatty()=False hardcodé + fileno() absent → Python en mode non-interactif → EOFError silencieuse → questionnaire avorté sans rien écrire.
Suppression complète du logging fichier et du _TeeStream
SESSION #9
update_markdown écrasait tout
Chaque write remplaçait le bloc ### entier. "France" → tout le profil disparaissait, remplacé par "France" brut.
Enrichissement si contenu réel existant, remplacement si placeholder uniquement
SESSION #10
Labels template = faux positifs
"- **Nom/Pseudo** : pas encore renseigné" contient "nom" → alias trouvé → champ considéré rempli → seulement 2 questions sur 5 détectées.
_line_is_real_value() : alias sur placeholder ou label sans valeur → ignoré
SESSION #11
Question style en boucle infinie
Clé "préférences" ne correspondait à aucun ## dans le fichier → section_content toujours vide → gap toujours détecté → reproposé à chaque session.
Fusionné sous "identité utilisateur" — la vraie section parente dans le fichier
SESSION #12
Header ### matchait comme alias réel
_extract_section_content() retourne tout en lowercase y compris ### préférences & style. L'alias "style" trouvé dans le titre → _line_is_real_value() → True → champ jamais détecté comme vide. Révélé par les tests unitaires.
Skip explicite des lignes # dans _detect_structural_gaps — les titres de sous-section ne sont plus analysés
SESSION #13
Merge écrasait les vraies valeurs
is_placeholder = True si UNE ligne du bloc contient un marqueur. Profil avec "Matt" + deux placeholders → merged = content → Matt effacé silencieusement. Révélé par les tests unitaires.
Filtrage ligne par ligne : real_lines = lignes sans marqueur. Merge = réelles + nouveau contenu. Valeurs existantes toujours préservées
